# Control Board v3 Architecture Spec

**Goal:** Make the code *sing* by clarifying ownership, centralizing playback intent, and separating compute from render. v3 preserves v2 user‑visible behavior while eliminating conflicting writers and scattered logic.

---

## 1. System Contract (v3)

**Inputs → Intent → Domain State → Display State → Render**

- **Inputs**: RP2040 UART events, HA/MA sensors, Arylic HTTP/TCP
- **Intent**: user intent (menu navigation, audio focus, lighting focus)
- **Domain State**: authoritative globals (audio activity, selected meta, screen mode)
- **Display State**: screen state machine (menu/now‑playing/lighting/screensaver/blank)
- **Render**: draw without side‑effects

---

## 2. Ownership Rules (Single Writer)

Each shared global has a **single owner**, and other modules must *request* changes via scripts or inputs.

| Global | Owner | Rule |
|---|---|---|
| `screensaver_active` | **Display state machine** | Only set in display lambda. System interval must not mutate. |
| `screen_mode` | **UI intent scripts** | Set by input intent (focus_audio/focus_lighting/menu). Display uses read‑only. |
| `menu_active`, `menu_level`, `menu_index` | **UI scripts** | Render reads only. |
| `last_input_ms` | **Input scripts** | UI/audio input scripts update; render never writes. |
| `playing_effective` (concept) | **Display predicate block** | Single canonical predicate used by UI/logging/screensaver. |

---

## 3. Centralized Playback Predicate

**Definition:** `playing_effective` is true when *any* source indicates active playback or a valid grace window applies.

**Signals:**
- HA/MA states (1–6)
- Arylic playing (TCP/HTTP + LR)
- Meta recency (grace + stale windows)
- Source active hints (arylic source != idle)

**Rule:** Only one block computes it. All other modules (logging, screensaver) *read* it.

---

## 4. Meta Selection Rules

**Meta selection** occurs once per compute cycle and yields:
- `meta_idx` (source slot)
- `meta_title`, `meta_artist`, `meta_album`, `meta_source`
- `meta_duration`, `meta_pos`, `meta_pos_ms`

**Rules preserved from v2:**
- Recency gates and stall checks
- Priority scoring: playing > active > freshness > priority bonus
- Override support via `meta_override_idx`

**Change in v3:** selection is computed in a **compute block**, not during render; rendering only consumes cached fields.

---

## 5. Display State Machine (Authoritative)

**States:**
- SPLASH → BOOT_MENU → MENU → NOW_PLAYING → LIGHTING → SCREENSAVER → BLANK

**Decision order (v3):**
1. Splash if active
2. Boot menu window
3. Menu active
4. Now Playing if `screen_mode==AUDIO` **and** `playing_effective || audio_wake_active`
5. Lighting if `screen_mode==LIGHTING` and lighting activity window
6. Screensaver if idle threshold exceeded
7. Blank otherwise

**Ownership:** Only this block sets `screensaver_active`.

---

## 6. Module Boundaries (v3)

**Audio package** (`control-board-audio.yaml`)
- Owns audio control inputs, Arylic TCP/HTTP, HA/MA metadata ingestion.
- Does *not* decide display state.

**UI package** (`control-board-ui.yaml`)
- Owns menu navigation and input intent.
- Sets `screen_mode` and menu globals only through user input.

**System package** (`control-board-system.yaml`)
- Owns globals, logging, intervals.
- Must **not** mutate display state (screensaver/menu). Logs should read canonical predicate.

**Peripherals (OLED)** (`control-board-peripherals-no-rings.yaml`)
- Owns display state machine and render.
- Computes playback predicate and meta selection **once per frame** (cached).

---

## 7. Render Contract

Render blocks:
- **No side effects** outside cached display fields and single ownership globals.
- No mutation of `screen_mode`, `menu_*`, `last_input_ms`.
- All state transitions are driven by the display state machine.

---

## 8. Verification (Architecture First)

- **Single writer per global** (no conflicting intervals or scripts).
- **Log/UI agreement** on playback state.
- **Now Playing** remains visible while audio is active or within grace window.
- **Screensaver** activates only when display state machine decides it.

---

## 9. Implementation Targets

**Phase 1:** `control-board-peripherals-no-rings.yaml`
- Create canonical predicate block.
- Convert screen flow to spec.
- Remove side‑effects in render.

**Phase 2:** `control-board-system.yaml`
- Remove screensaver mutation from interval.
- Update status logging to use canonical predicate.

**Phase 3:** `control-board-ui.yaml` + `control-board-audio.yaml`
- Ensure menu/screen_mode are only written by intent scripts.

**Phase 4:** Mirror to `control-board-peripherals.yaml` after no‑rings stabilizes.
